# 寒影 · HanHub 开源致谢

本项目受到以下优秀开源项目的启发和滋养，在此致以诚挚的感谢（按字母排序）：

## 静态站点生成器

| 项目 | 致谢 |
|------|------|
| **Astro** | 岛屿架构和零 JavaScript 默认的理念深刻影响了 HanHub 的构建思路 |
| **Gatsby** | GraphQL 数据层的设计让我们重新思考内容与数据的关系 |
| **Gridsome** | Vue 生态下的静态生成方案启发了多框架支持的设计 |
| **Hexo** | 中文社区最成熟的博客框架，插件生态为云笺平台提供重要参考 |
| **Hugo** | 极快的构建速度和灵活的内容组织方式，是性能优化的标杆 |
| **Jekyll** | GitHub Pages 原生支持的鼻祖，奠定了静态博客的基石 |
| **Next.js** | ISR 和混合渲染策略启发了我们关于动态与静态的平衡思考 |
| **Nuxt.js** | Vue 全栈框架的模块化设计为后台系统架构提供灵感 |
| **Pelican** | Python 社区优雅的静态生成器，展示了跨语言生态的可能性 |
| **Remix** | Web 标准的极致践行，启发了 HanHub 对 Web 基础能力的重视 |
| **SvelteKit** | 编译时优化的激进思路，让我们重新审视框架运行时成本 |

## CMS 与内容管理

| 项目 | 致谢 |
|------|------|
| **Decap CMS** | Git-based CMS 的典范，云笺在线编辑器的交互设计深受启发 |
| **Directus** | 灵活的 API 层设计和数据建模能力启发了后台管理系统的架构 |
| **Ghost** | 专业博客平台的会员订阅与内容分发模式为云笺 Pro 版提供参考 |
| **Grav** | 扁平文件 CMS 的理念验证了无数据库架构在博客场景的可行性 |
| **Strapi** | 可定制的后台 UI 和插件系统设计直接影响了管理后台的交互 |
| **WordPress** | 无可争议的 CMS 之王，其主题与插件生态是云笺市场的长期目标 |

## 博客平台与社区

| 项目 | 致谢 |
|------|------|
| **Discourse** | 现代化社区论坛的设计语言影响了 HanHub 整体的 UI 质感 |
| **Emlog** | 国产轻量博客的先驱，"每篇文章都值得记录"的理念与 HanHub 共鸣 |
| **Halo** | 国产博客生态的革新者，现代化的后台体验为墨韵后台指明方向 |
| **Qexo** | 多平台统一的博客管理理念启发了云笺的多生成器支持设计 |
| **Typecho** | 极致的轻量与高性能，证明博客不需要臃肿的架构 |
| **VanBlog** | 简约而不简单的理念与 HanHub "回归写作" 的追求一致 |
| **WriteFreely** | 去中心化写作平台启发了我们对数据所有权的思考 |
| **Z-Blog** | 中文博客社区的常青树，见证了国内独立博客的发展历程 |

## 特别感谢

感谢所有选择了独立博客这条路的写作者。每一个独立站点都是互联网去中心化浪潮中的一个节点，HanHub 希望能让更多人轻松地拥有自己的数字角落。
